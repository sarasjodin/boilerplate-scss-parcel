More About My "Sassy Website"

This project is part of the Web Development program at Mid Sweden University and involves creating a website using Node.js, NPM, Parcel, and Sass (SCSS).

Purpose of the Assignment

  • Understand the purpose of CSS preprocessors.
  • Learn how to use SASS to simplify CSS development.
  • Practically apply the development environment set up in previous modules.
  • Be able to automate the conversion of SASS files using Parcel.

Minimum Requirements

  • A simple, fictional website with at least five menu options (not all need to be functional).
  • At least two HTML pages (a homepage and a subpage about Sass).
  • Based on a Parcel boilerplate, but without reusing old CSS files.
  • Use of variables (at least the color scheme should be stored as variables).
  • Color functions (at minimum, `darken` and `lighten` should be used, but more are encouraged).
  • Nesting, Mixins, and Selector Inheritance (using the `@extend` keyword).
  • If/else statements, at least one partial, and a well-structured CSS file organization.
  • Deployment should not use FTP/SFTP hosting but instead platforms such as GitHub Pages, Netlify, Vercel, or Render.

My Evaluation

Sass extends the functionality of regular CSS by introducing powerful features that make styling more efficient, maintainable, and scalable. Some key advantages and disadvantages of using Sass include:

Pros of the CSS Preprocessor Sass

  • Better Organization
    Using partials and imports allows for a structured and modular CSS approach.
  • Variables
    Define colors, font sizes, and other values in one place and reuse them throughout the styles.
  • Mixins
    Write reusable blocks of CSS to avoid code repetition and improve maintainability.
  • Selector Inheritance (`@extend`)
    Helps in reducing redundancy by allowing shared styles between elements.
  • Functions & Nesting
    Improves readability and makes it easier to write CSS with logical hierarchy.
  • Math Operations
    Enables calculations directly in CSS, such as adjusting padding or font sizes dynamically.

Cons of the CSS Preprocessor Sass

  • Learning Curve
    While Sass simplifies many CSS tasks, beginners may find it overwhelming at first.
  • Compilation Required
    Unlike standard CSS, Sass needs to be compiled into regular CSS before it can be used in the browser. Parcel is taking care of that for this website.
  • Can Encourage Overuse
    Nesting too deeply can lead to overly specific and difficult-to-maintain styles.
  • Tooling Dependency
    Requires Node.js, npm, and build tools like Parcel or Webpack, which adds complexity.

How Was It to Get Started Working with a CSS Preprocessor?

The Initial Challenge

Transitioning from regular CSS to Sass was initially overwhelming due to the new syntax and additional setup required. Instead of taking small steps, I decided to dive straight into the 7-1 pattern method, which is quite complex for someone without much prior Sass experience.

The Learning Curve

As I became more comfortable with variables, mixins, and nesting, I quickly realized the immense benefits of Sass. The ability to store and reuse values significantly reduced redundancy, and nesting made my stylesheets more structured and readable.

Boosting Efficiency with Automation

One of the biggest game-changers was automating the Sass compilation process using Parcel. This made it seamless to integrate Sass into my workflow, eliminating the need for manual CSS updates and ensuring that all styles were efficiently processed.

The Power of Structured Styling

The fact that multiple stylesheets can be structured and managed through a single main.scss file has made my development process much smoother. By organizing my styles into different partials, I can maintain a clean and scalable codebase, making future updates and modifications effortless.
